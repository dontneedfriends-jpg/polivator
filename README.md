# Polivator — автоматический полив растений на ESP32-S3

Проект модульной системы мониторинга и автоматического полива почвы на базе
ESP32-S3 с e-ink дисплеем 2.13", веб-интерфейсом и индивидуальной калибровкой
каждого датчика. Все настройки хранятся в энергонезависимой памяти (NVS).

## Возможности

- **До 5 ёмкостных датчиков влажности почвы** на любых свободных GPIO ESP32-S3.
- **Индивидуальная калибровка** каждого датчика: значения «сухо»/«влажно», имя,
  GPIO-пин, включение/выключение.
- **E-ink дисплей 2.13"** (250×122, чёрно-бело-красный) с частичным обновлением
  для быстрого отображения без полного мерцания.
- **Веб-интерфейс в стиле IBM Carbon** (тёмная/светлая тема, круговые
  индикаторы, таблицы пинов, расширенные настройки) на любом устройстве
  в локальной сети.
- **Server-Sent Events (SSE)** для live-обновления dashboard без перезагрузки.
- **REST API** для интеграции с Home Assistant, Node-RED, MQTT-бриджами и т.д.
- **OTA-обновление** прошивки через веб-интерфейс или по локальной сети.
- **Управление насосом** через GPIO + опциональный автоматический режим
  (включает насос при падении влажности ниже порога).
- **Настройка WiFi** через captive portal при первом запуске.
- **Гибкие параметры**: интервалы чтения и обновления, стратегия обновления,
  режимы сна (off / light / deep), единицы отображения (% / raw / V),
  количество усредняемых выборок АЦП.
- **Серийная калибровка** через USB-Serial без подключения к веб-интерфейсу.

## Аппаратные требования

| Компонент | Модель / характеристики |
|-----------|------------------------|
| Микроконтроллер | ESP32-S3-WROOM-1 (8 MB Flash, USB-C) |
| Дисплей | WeActStudio 2.13" e-ink, SSD1680 / GDEY0213Z98 (250×122, B/W/R) |
| Датчики | 1–5 ёмкостных датчиков влажности почвы с аналоговым выходом 0–3.3 В |
| Насос | 12 В перистальтический / диафрагменный насос + MOSFET-ключ или реле |
| Питание | USB 5 В (≥ 1 А) или Li-ion + стабилизатор 3.3 В |
| Дополнительно | Провода, макетная плата или 3D-печатный корпус, конденсаторы 100 нФ |

### Используемые пины ESP32-S3

| GPIO | Назначение |
|------|-----------|
| 4    | E-ink BUSY |
| 5    | E-ink CS |
| 16   | E-ink RST |
| 17   | E-ink DC |
| 18   | E-ink SCK (SPI) |
| 21   | E-ink MOSI (SPI) |
| 38   | Реле насоса (можно любой свободный GPIO) |
| 1, 3 | UART0 (Serial) — не занимать под датчики |
| 6–10 | Внутренняя SPI flash — **нельзя** использовать |
| 2, 12, 13, 14, 15 | Рекомендуемые ADC-пины для датчиков |

### Программные модули

| Модуль | Назначение |
|--------|-----------|
| `SensorManager` | Чтение сырых значений с АЦП, вычисление процента и напряжения |
| `Calibration` | Хранение калибровок (dry/wet), названий и пинов в NVS как JSON |
| `Display` | Драйвер e-ink дисплея с частичным/полным обновлением, отдельная FreeRTOS-задача |

| `WaterPump` | Управление реле, авто-режим, калибровка расхода (мл/с) |
| `WebServer` | HTTP/REST API, SSE-события, captive portal для WiFi, раздача UI |

## Сборка устройства

### Что понадобится

1. **Плата ESP32-S3** с 8 MB Flash и USB-C (например, ESP32-S3-DevKitC-1).
2. **E-ink дисплей WeActStudio 2.13"** с контроллером SSD1680 (чёрно-бело-красный).
3. **5 ёмкостных датчиков влажности почвы** (например, Capacitive Soil Moisture Sensor v1.2).
4. **Перистальтический насос 12 В** или 5 В диафрагменный (KPHM-100 и т.п.).
5. **MOSFET-ключ** (IRLZ44N, AO3400) или **5 В реле** (SRD-05VDC-SL-C).
6. **Провода Dupont**, макетная плата, клеммники.
7. **Блок питания 12 В / 1 А** (для насоса) или USB-источник ≥ 2 А.
8. **Понижающий DC-DC преобразователь 12 В → 3.3 В** (если нет USB).

### Пошаговая инструкция

#### 1. Подготовка ESP32-S3

Установите драйверы CH340/CP2102 (если используете клон) или используйте
плату с нативным USB. Подключите плату к ПК, убедитесь, что в диспетчере
устройств появился COM-порт (например, `COM3` / `COM4`).

#### 2. Подключение e-ink дисплея

Соедините дисплей с ESP32-S3 по SPI (используйте короткие провода ≤ 10 см):

| Пин дисплея | → | GPIO ESP32-S3 |
|-------------|---|---------------|
| VCC         | → | 3.3V         |
| GND         | → | GND          |
| BUSY        | → | GPIO 4       |
| RES / RST   | → | GPIO 16      |
| D/C         | → | GPIO 17      |
| CS          | → | GPIO 5       |
| SCK / SCL   | → | GPIO 18      |
| MOSI / SDA  | → | GPIO 21      |

#### 3. Подключение датчиков влажности

Для каждого датчика:

| Пин датчика | → | ESP32-S3 |
|-------------|---|----------|
| VCC         | → | 3.3V     |
| GND         | → | GND      |
| AOUT        | → | свободный ADC-пин (например, GPIO 2, 12, 13, 14, 15) |

**Рекомендация:** припаяйте керамический конденсатор 100 нФ между VCC и GND
каждого датчика как можно ближе к плате — это сильно уменьшает шум АЦП.

> ⚠️ **Важно:** не подключайте датчики к GPIO 1/3 (Serial), GPIO 4/5
> (дисплей), GPIO 6–10 (внутренняя flash — сожжёте прошивку), GPIO 11–20
> (ADC2 — нестабильно при активном WiFi).

#### 4. Подключение насоса

Подключите MOSFET или реле между ESP32-S3 и насосом:

```
ESP32 GPIO 38 ──► Gate (MOSFET) / IN (модуль реле)
                  Drain → Source → GND
                            │
                       Насос (−)
                       
Блок питания 12 В (+) ──► Насос (+)
Блок питания 12 В (−) ──► GND (общий с ESP32)
```

**Защита от индуктивных выбросов:** параллельно насосу установите диод
1N4007 (катодом к +12 В).

#### 5. Питание

- **Простой вариант:** USB-кабель 5 В / ≥ 1 А от старого зарядного.
  Насос подключите к отдельному 12 В блоку питания.
- **Автономный вариант:** Li-ion 18650 + TP4056 модуль заряда + DC-DC 3.3 В.
  Насос питается через повышающий DC-DC до 12 В.

#### 6. (Опционально) Корпус

- Напечатайте корпус на 3D-принтере (модели для Polivator можно найти
  в разделе Releases).
- Или используйте пластиковую коробку 100×60×40 мм с отверстиями для
  дисплея, датчиков и кабеля насоса.

#### 7. Финальная проверка перед прошивкой

- [ ] Все контакты VCC / GND подключены и нет коротких замыканий.
- [ ] Дисплей показывает «чистый» белый/красный (без артефактов) при подаче питания.
- [ ] Датчики дают стабильные показания 1500–3500 при комнатной влажности.
- [ ] USB-кабель подключён к ПК, COM-порт определяется.

## Прошивка

### Требования

- [PlatformIO](https://platformio.org/) (расширение VS Code или CLI)
- Python 3.7+
- Драйверы USB-to-Serial (CH340, CP2102 или нативный USB)

### Установка

```bash
# 1. Клонировать репозиторий
git clone https://github.com/dontneedfriends-jpg/polivator.git
cd polivator

# 2. (опционально) отредактировать src/main.cpp — массив sensorPins:
#    const uint8_t sensorPins[MAX_SENSORS] = {2, 12, 13, 14, 15};
#    const uint8_t PUMP_RELAY_PIN = 38;
#    По умолчанию — настройки для типовой разводки.

# 3. Собрать
pio run

# 4. Загрузить через USB
pio run -t upload

# 5. Открыть Serial Monitor (115200 бод)
pio device monitor
```

### OTA-обновление

После первой прошивки через USB можно обновлять по воздуху:

1. В `platformio.ini` раскомментируйте и укажите IP устройства:
   ```ini
   upload_protocol = espota
   upload_port = 192.168.1.42
   ```
2. Запустите:
   ```bash
   pio run -t upload
   ```

Или через веб-интерфейс: **Settings → Firmware Update → выберите .bin файл**.

## Первый запуск и настройка WiFi

1. Подключите ESP32 к USB. Устройство стартует в режиме точки доступа
   с SSID **`Polivator-Config`** (без пароля).
2. С телефона/ПК подключитесь к этой сети.
3. Откройте в браузере [http://192.168.4.1](http://192.168.4.1).
4. Введите SSID и пароль вашей домашней WiFi, нажмите **Save**.
5. Устройство перезагрузится и подключится к домашней сети.
6. IP-адрес можно узнать из Serial Monitor (115200 бод) или из админ-панели роутера.

**Доступ к веб-интерфейсу** защищён HTTP Basic Auth:
- Логин: `admin`
- Пароль: `admin`

> Совет: после первого подключения смените пароль в `WebServer.cpp` (константы
> `DEFAULT_AUTH_USERNAME` / `DEFAULT_AUTH_PASSWORD`) и пересоберите прошивку.

## Веб-интерфейс и REST API

### Вкладки UI

- **Dashboard** — карточки датчиков с круговыми индикаторами влажности, метриками
  Raw / Voltage, кнопками калибровки (Dry / Wet / Reset) и полем для
  настройки GPIO-пина.
- **Pinout** — таблицы подключения дисплея и датчиков.
- **Settings** — WiFi, интервалы чтения и обновления, стратегия обновления,
  режим сна, единицы отображения, OTA, отладка, автополив (autoMode / threshold).

### REST API

Все эндпоинты требуют Basic Auth (admin / admin по умолчанию).

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `GET`    | `/api/status` | Статус всех датчиков (JSON) |
| `GET`    | `/api/calibration` | Текущие настройки калибровки |
| `POST`   | `/api/calibrate/dry?index=N` | Установить текущее значение как «сухо» для датчика N |
| `POST`   | `/api/calibrate/wet?index=N` | Установить текущее значение как «влажно» для датчика N |
| `POST`   | `/api/calibrate/reset?index=N` | Сбросить калибровку датчика N |
| `POST`   | `/api/config/sensor?index=N` | `{"name": "Горшок 1"}` или `{"pin": 12}` |
| `DELETE` | `/api/sensor?index=N` | Отключить датчик N |
| `POST`   | `/api/sensor/enable?index=N` | Включить датчик N |
| `GET / POST` | `/api/settings` | Получить / сохранить системные настройки |
| `POST`   | `/api/config/wifi` | Сохранить SSID / пароль WiFi |
| `POST`   | `/api/pump/run` `{"seconds": 5}` | Запустить насос на 5 секунд |
| `POST`   | `/api/pump/stop` | Остановить насос |
| `GET`    | `/api/pump` | Статус насоса + настройки |
| `POST`   | `/api/pump/config` `{"autoMode": true, "threshold": 30, "flowRate": 1.5}` | Настройки насоса |
| `POST`   | `/api/pump/calibrate` `{"seconds": 10, "measuredMl": 50}` | Калибровка расхода |
| `GET`    | `/events` | Server-Sent Events с обновлениями в реальном времени |
| `POST`   | `/update` | OTA-обновление прошивки (multipart) |

### Интеграция с Home Assistant

Добавьте в `configuration.yaml`:

```yaml

  - resource: http://192.168.1.42/api/status
    username: admin
    password: admin
    scan_interval: 30
    sensor:
      - name: Polivator Sensor 1
        value_template: "{{ value_json.sensors[0].percentage }}"
        unit_of_measurement: "%"
```

## Калибровка датчиков

Ёмкостные датчики дают «сырое» значение АЦП, которое зависит от модели датчика,
длины кабеля и влажности почвы. Для перевода в проценты нужна калибровка.

### Через веб-интерфейс

1. Откройте вкладку **Dashboard**.
2. Выберите датчик (S1–S5).
3. Поместите датчик в **сухую** среду (воздух) и нажмите **Dry** — текущее
   значение запомнится как «полностью сухо» (0%).
4. Поместите датчик в **воду** или очень мокрую почву, нажмите **Wet** —
   значение запомнится как «полностью влажно» (100%).
5. Повторите для каждого датчика.

### Через Serial Monitor

1. Откройте Serial Monitor на 115200 бод.
2. Отправьте `c` — войти в режим калибровки.
3. Отправьте `0`–`4` — выбрать датчик.
4. Команды:
   - `d` — установить сухой порог
   - `w` — установить влажный порог
   - `r` — сбросить калибровку
   - `s` — показать текущий статус всех датчиков

## Системные настройки

Все параметры управляются через вкладку **Settings** или REST API
(`/api/settings`).

| Параметр | Диапазон | Описание |
|----------|----------|----------|
| Read Interval | 1–3600 с | Как часто считываются показания датчиков |
| ADC Samples | 1–100 | Количество выборок АЦП для усреднения (больше — точнее, но медленнее) |
| Web Update Interval | 1–3600 с | Интервал SSE-событий для веб-интерфейса |
| Display Refresh Interval | 1–3600 с | Как часто обновляется e-ink дисплей |
| Update Priority | 0/1/2 | Стратегия обновления: time / change / balanced |
| Sleep Mode | 0/1/2 | Disabled / light sleep / deep sleep |
| Display Unit | 0/1/2 | Единица отображения: % / raw ADC / voltage |
| OTA Enabled | bool | Включить OTA-обновления |
| Debug Enabled | bool | Включить отладочный вывод в Serial |

Все настройки автоматически сохраняются в NVS через 2 секунды после
последнего изменения (отложенная запись для уменьшения износа Flash).

## Расширение и доработка

### Добавление 6-го и более датчиков

Измените `MAX_SENSORS` в `lib/Common/Types.h`, добавьте пины в `sensorPins`
в `main.cpp` и обновите UI в `WebServer/index.h`.

### Подключение датчика температуры почвы

Добавьте в `SensorManager` новый метод `readTemperature(uint8_t index)`,
который читает с DS18B20 (OneWire) или DHT22. Выведите в `/api/status`.

### MQTT вместо/вместе с REST

Добавьте `lib/Mqtt/` с модулем, который публикует `polivator/sensor/N/percent`
при каждом обновлении (через SSE-событие).

### Более мощный насос

Замените `digitalWrite` на `ledcWrite` для плавного ШИМ-управления
производительностью насоса. Это особенно полезно для перистальтических
насосов, где важна точная дозировка.

### Корпус 3D-печать

Модели корпуса можно сделать в OpenSCAD или Fusion 360. Внешние размеры
зависят от выбранного e-ink дисплея и количества датчиков.

## Устранение неисправностей

| Симптом | Вероятная причина | Решение |
|---------|-------------------|---------|
| Дисплей ничего не показывает | Неправильное подключение SPI | Проверьте CS, DC, RST, BUSY, MOSI, SCK |
| Дисплей показывает мусор | Плохой контакт, длинные провода | Используйте короткие провода ≤ 10 см, проверьте пайку |
| Датчик всегда 4095 | Обрыв сигнального провода или датчик не запитан | Проверьте 3.3 В на VCC датчика, прозвоните AOUT |
| Датчик всегда 0 | КЗ на сигнальной линии или пин = ADC2 при WiFi | Используйте только пины 2, 12, 13, 14, 15 |
| Показания «прыгают» | Наводки от WiFi / насоса | Добавьте 100 нФ конденсатор на датчик, усредните `adcSamples` |
| Не подключается к WiFi | Неверный SSID / пароль | Подключитесь к AP `Polivator-Config`, введите заново |
| Веб-интерфейс не открывается | Устройство не получило IP | Смотрите Serial Monitor, проверьте DHCP роутера |
| OTA не работает | OTA отключён в Settings | Включите `otaEnabled = true` в Settings или `/api/settings` |
| Насос не включается | Реле инвертировано или нет питания 12 В | Проверьте `activeHigh` в `WaterPump::begin()` |
| Калибровка сбрасывается | Повреждение NVS | `pio run -t erase` (удалит ВСЕ данные) и заново настройте |
| Устройство перезагружается | Сработал WDT, плохое питание | Используйте USB-кабель с данными, БП ≥ 1 А |

### Полный сброс

```bash
pio run -t erase      # стереть всю Flash
pio run -t upload     # загрузить заново
```
