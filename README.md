# Датчик влажности почвы (Polivator)

Модульная система мониторинга влажности почвы на ESP32-S3 с поддержкой до 5 ёмкостных датчиков, e-ink дисплеем WeActStudio 2.13", веб-интерфейсом в стиле IBM Carbon и индивидуальной калибровкой каждого датчика. Все настройки сохраняются в энергонезависимой памяти.

## Возможности

- Поддержка до 5 ёмкостных датчиков влажности почвы.
- Для каждого датчика: название, GPIO-пин, калибровка (сухо/влажно), включение/выключение.
- Настраиваемые пины датчиков прямо из веб-интерфейса.
- Отображение влажности на e-ink дисплее с частичным и быстрым обновлением экрана.
- Веб-интерфейс в стиле IBM Carbon: тёмная/светлая тема, круговые индикаторы, таблицы пинов, расширенные настройки.
- Server-Sent Events (SSE) для live-обновления dashboard.
- Калибровка через веб-интерфейс или последовательный порт.
- WiFi: точка доступа + captive portal для первичной настройки, затем подключение к домашней сети.
- OTA-обновление прошивки через веб-интерфейс.
- REST API для удалённого управления и мониторинга.
- Расширенные настройки: интервалы чтения/обновления, стратегия обновления, режимы сна, единицы отображения, количество ADC-выборок.

## Аппаратные требования

- ESP32-S3 (WROOM-1) — рекомендуется плата с USB-C и 8+ MB Flash.
- WeActStudio 2.13" e-ink дисплей (SSD1680 / GDEY0213Z98, чёрно-бело-красный, 250×122).
- 1–5 ёмкостных датчиков влажности почвы с аналоговым выходом 0–3.3 В.
- Соединительные провода, макетная плата.
- Питание: USB 5V или аккумулятор + стабилизатор 3.3 В.

## Важное замечание о пинах датчиков

На ESP32-S3 **не все GPIO можно использовать** для АЦП одновременно с дисплеем и WiFi:

| Пины | Ограничение |
|------|-------------|
| GPIO1, GPIO3 | Используются Serial/UART (если Serial активен). |
| GPIO4, GPIO5 | Заняты e-ink дисплеем (BUSY и CS). |
| GPIO6–GPIO10 | Внутренняя flash, **использовать нельзя**. |
| GPIO11–GPIO20 | ADC2. Чтение при включённом WiFi может давать ошибки. |
| GPIO2, GPIO12–GPIO15, GPIO38–GPIO48 | Рекомендуемые свободные пины (если доступны на плате). |

По умолчанию в прошивке заданы пины `GPIO1–5`, но из-за конфликтов с дисплеем и Serial датчики могут не работать. **Обязательно переназначьте пины через веб-интерфейс** (Dashboard → поле GPIO в карточке датчика) или измените массив `sensorPins` в `src/main.cpp` перед прошивкой.

## Подключение пинов

### Дисплей (WeActStudio 2.13" e-ink)

| Пин дисплея | GPIO ESP32-S3 |
|-------------|---------------|
| BUSY        | 4             |
| RES / RST   | 16            |
| D/C         | 17            |
| CS          | 5             |
| SCK / SCL   | 18            |
| MOSI / SDA  | 21            |
| GND         | GND           |
| VCC         | 3.3V          |

### Датчики влажности (1–5 шт.)

| Пин датчика | ESP32-S3 |
|-------------|----------|
| AOUT (сигнал) | настраивается в веб-интерфейсе или `sensorPins` |
| VCC         | 3.3V (общий для всех) |
| GND         | GND (общий) |

Рекомендуется керамический конденсатор 100 нФ между VCC и GND каждого датчика для подавления шумов.

## Настройка программного обеспечения

1. Установите [PlatformIO](https://platformio.org/) в VS Code или используйте CLI.
2. Клонируйте репозиторий:
   ```bash
   git clone <url>
   cd polivator
   ```
3. Зависимости установятся автоматически при сборке: `GxEPD2`, `ESPAsyncWebServer`, `AsyncTCP`, `ArduinoJson`.
4. При необходимости отредактируйте `src/main.cpp`, массив `sensorPins` под свои пины.
5. Соберите и загрузите:
   ```bash
   pio run -t upload
   ```
6. Монитор порта:
   ```bash
   pio device monitor
   ```

### OTA-обновление

После первой загрузки через USB прошивку можно обновлять по воздуху:

1. Укажите IP-адрес устройства в `platformio.ini`:
   ```ini
   upload_protocol = espota
   upload_port = 192.168.1.114
   ```
2. Загрузите:
   ```bash
   pio run -t upload
   ```

Или используйте кнопку OTA в веб-интерфейсе.

## Первый запуск / Настройка WiFi

1. Устройство стартует в режиме точки доступа с SSID `Polivator-Config`.
2. Подключитесь телефоном/ПК к этой WiFi.
3. Откройте в браузере [http://192.168.4.1](http://192.168.4.1).
4. Введите SSID и пароль вашей домашней сети, нажмите **Save**.
5. Устройство перезагрузится и подключится к домашней сети.
6. IP-адрес устройства можно узнать из Serial Monitor или роутера.

Доступ к веб-интерфейсу защищён базовой авторизацией:
- Логин: `admin`
- Пароль: `admin`

## Веб-интерфейс

Откройте IP устройства в браузере. Интерфейс выполнен в стиле IBM Carbon и содержит три вкладки:

- **Dashboard** — карточки датчиков с круговыми индикаторами влажности, метриками Raw/Voltage, кнопками калибровки и полем для настройки GPIO-пина.
- **Pinout** — таблицы подключения дисплея и датчиков.
- **Settings** — WiFi, интервалы чтения/обновления, стратегия обновления, режим сна, единицы отображения, OTA, отладка.

### REST API

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/api/status` | Статус всех датчиков (JSON) |
| GET | `/api/calibration` | Текущие настройки калибровки |
| POST | `/api/calibrate/dry?index=N` | Установить текущее значение как сухой порог для датчика N |
| POST | `/api/calibrate/wet?index=N` | Установить текущее значение как влажный порог для датчика N |
| POST | `/api/calibrate/reset?index=N` | Сбросить калибровку датчика N на значения по умолчанию |
| POST | `/api/config/sensor?index=N` | Установить название (`{"name": "Sensor1"}`) или пин (`{"pin": 12}`) датчика N |
| DELETE | `/api/sensor?index=N` | Отключить датчик N |
| POST | `/api/sensor/enable?index=N` | Включить датчик N |
| GET / POST | `/api/settings` | Получить/сохранить системные настройки |
| POST | `/api/config/wifi` | Сохранить SSID и пароль WiFi |
| GET | `/events` | Server-Sent Events с обновлениями в реальном времени |
| POST | `/update` | OTA-обновление прошивки |

**Пример ответа `/api/status`:**
```json
{
  "sensors": [
    {"name": "Горшок 1", "pin": 12, "raw": 2048, "percentage": 45, "voltage": 1.65, "dry": 3000, "wet": 1200, "enabled": true},
    {"name": "Горшок 2", "pin": 13, "raw": 1800, "percentage": 60, "voltage": 1.45, "dry": 3000, "wet": 1200, "enabled": true}
  ],
  "wifi": "connected"
}
```

**Пример тела `/api/settings`:**
```json
{
  "readInterval": 2,
  "displayInterval": 30,
  "webInterval": 2,
  "updatePriority": 0,
  "sleepMode": 0,
  "otaEnabled": true,
  "debugEnabled": true,
  "displayUnit": 0,
  "adcSamples": 10
}
```

## Настройки

Все настройки управляются через вкладку **Settings** и сохраняются в памяти устройства.

| Параметр | Диапазон | Описание |
|----------|----------|----------|
| Read Interval | 1–3600 с | Как часто считываются значения с датчиков. |
| ADC Samples | 1–100 | Количество выборок АЦП, усредняемых для одного измерения. Больше — точнее, но медленнее. |
| Web Update Interval | 1–3600 с | Интервал SSE-событий для веб-интерфейса. |
| Display Refresh Interval | 1–3600 с | Как часто обновляется e-ink дисплей. |
| Update Priority | 0/1/2 | Стратегия обновления: time priority / change priority / balanced. |
| Sleep Mode | 0/1/2 | Disabled / light sleep / deep sleep. Deep sleep отключает веб-интерфейс. |
| Display Unit | 0/1/2 | Единица отображения: % / raw ADC / voltage. |
| OTA Enabled | bool | Включить обновление прошивки по воздуху. |
| Debug Enabled | bool | Включить отладочный вывод в Serial. |

## Калибровка

### Через веб-интерфейс

1. На вкладке **Dashboard** выберите датчик.
2. Поместите датчик в сухую среду (воздух), нажмите **Dry**.
3. Поместите датчик в воду или мокрую почву, нажмите **Wet**.
4. Повторите для каждого датчика.

### Через последовательный порт

- Войдите в режим калибровки: **c**
- Выберите датчик цифрой **0..4**
- Команды:
  - **d** — установить сухой порог для выбранного датчика
  - **w** — установить влажный порог
  - **r** — сбросить калибровку для выбранного датчика
  - **s** — показать текущий статус всех датчиков

## Обновление дисплея

Дисплей использует частичное обновление (`setPartialWindow`) для быстрого изменения только нужной области без полного мерцания экрана. Каждое 10-е обновление выполняется полным (`setFullWindow`) для очистки возможного остаточного изображения (ghosting). После каждого обновления дисплей переводится в режим пониженного энергопотребления (`powerOff`).

## Расширение до автополива

Проект спроектирован модульно. Для добавления автополива:

1. Добавьте модуль `WaterPump` с классом для управления реле/клапаном через GPIO.
2. Добавьте модуль `Scheduler` для настройки расписания полива.
3. В `WebServer` добавьте эндпоинты для управления насосом и расписанием.
4. В `main.cpp` добавьте логику: если процент влажности < порога → включить насос на N секунд.

## Поиск неисправностей

| Проблема | Решение |
|----------|--------|
| Дисплей не показывает изображение | Проверьте SPI подключение (CS, DC, RST, BUSY, MOSI, SCK). Убедитесь, что питание 3.3 В. |
| Датчик всегда показывает 4095 или 0 | Проверьте питание датчика и правильность GPIO-пина. Убедитесь, что пин не занят дисплеем или Serial. |
| Датчики не отображаются в веб-интерфейсе | Проверьте, что датчики включены (`enabled = true`) и назначены свободные ADC-пины. |
| Не удаётся подключиться к WiFi | Убедитесь в правильности SSID и пароля. Сбросьте настройки WiFi через `pio run -t erase`. |
| Калибровка не сохраняется после перезагрузки | Проверьте, что NVS не заполнен. Стирайте флеш через `pio run -t erase`. |
| Веб-интерфейс не загружается | Убедитесь, что устройство получило IP (смотрите Serial Monitor). Проверьте firewall. |
| Обновление дисплея мерцает | Это норма для e-ink. Частичное обновление уменьшает мерцание; полное обновление раз в 10 циклов очищает ghosting. |
